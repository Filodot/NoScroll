# NoScrol — документация MVP

NoScrol — Android-приложение для осознанного ограничения YouTube Shorts. Репозиторий содержит работающий Android-прототип и сопровождающую продуктовую, UX/UI, техническую, privacy и QA-документацию.

## Статус

- Этап: работающий локальный прототип MVP.
- Целевая платформа: Android.
- Первая поддерживаемая поверхность: Shorts в нативном приложении YouTube.
- Рабочее название: `NoScrol`, не является финальным брендом.
- Язык MVP: русский; архитектура и детектор должны быть готовы к локализации.

## Локальная разработка

Требования: JDK 21 и Android SDK 37. Основная проверка проекта:

```powershell
.\gradlew.bat check assembleDebug
```

Debug APK после успешной сборки находится в `app/build/outputs/apk/debug/`.

## Зафиксированная конфигурация MVP

| Параметр | Рекомендованное значение | Диапазон настройки |
|---|---:|---:|
| Интервал Shorts до задания | 5 минут | 1–30 минут |
| Общий дневной лимит YouTube | 45 минут | 10–240 минут или выключен |
| Доступ после правильного ответа | равен интервалу Shorts | отдельно не настраивается в MVP |
| Сброс дневного лимита | локальная полночь | не настраивается в MVP |
| Emergency Stop | выключен | включается до ручного отключения |

## Главные продуктовые правила

- Время Shorts считается только тогда, когда нативный YouTube находится на активном экране и детектор уверенно распознал Shorts.
- Интервал Shorts накапливается между входами: выход из Shorts не обнуляет оставшееся время.
- После истечения интервала текущий Short не перекрывается: блокировка срабатывает на следующем подтверждённом свайпе.
- Если YouTube не сообщил событие свайпа, просмотр принудительно останавливается не позднее чем через 2 минуты после лимита.
- Задание всегда решается внутри NoScrol. Пока оно не решено, повторный вход в Shorts закрывается.
- Перед первым входом пользователь открывает NoScrol и решает лёгкое входное задание.
- После успешного задания пользователь получает новый полный интервал Shorts.
- Дневной лимит относится ко всему нативному приложению YouTube, включая обычные видео, но не веб-версию.
- После исчерпания дневного лимита задача не выдаёт дополнительное время. Доступ возможен только после следующего дневного сброса или через Emergency Stop.
- Emergency Stop отключает задания и дневной блок до ручного выключения. Перед включением обязательно вводится причина; пассивный учёт времени продолжается.
- MVP использует только локальные арифметические задания. Нейросеть, backend, аккаунты и облачная синхронизация исключены.
- Содержимое экрана YouTube не сохраняется и не передаётся с устройства.

## Термины

- **Интервал Shorts** — количество активного времени в Shorts между двумя заданиями.
- **Пауза / gate** — состояние закрытого доступа: YouTube покидается, а задание открывается внутри NoScrol.
- **Дневной лимит** — максимальное активное время всего приложения YouTube за локальные календарные сутки.
- **Emergency Stop** — пользовательский режим полного логического отключения ограничений до ручного возврата.
- **Fail open** — при технической неопределённости приложение не блокирует YouTube, чтобы избежать ложного срабатывания.
