# Умные ИИ-уроки — архитектура

## 1. Цель

Добавить в NoScroll последовательные образовательные курсы, которые создаются по теме или
пользовательским материалам, повторяют слабые понятия и выдают короткий урок как условие
разблокировки отвлекающего приложения.

ИИ никогда не вызывается в критическом пути блокировки. Приложение заранее хранит минимум три
проверенных урока. При отсутствии сети используются повторение или обычная локальная задача.

## 2. Принятые продуктовые решения

- приложение предназначено для локального частного использования без регистрации;
- Android-устройство выполняет маршрутизацию запросов, но не запускает LLM;
- содержимое материалов разрешено отправлять внешним AI-провайдерам;
- первая версия поддерживает PDF, DOCX, TXT, Markdown и ручной текст;
- курсы по теме используют доступные базы знаний, а при их отсутствии явно маркируются как
  основанные только на знаниях модели;
- программирование начинается с Python и SQL;
- для разблокировки нужен один правильный ответ;
- подозрительное задание заменяется без штрафа и без изменения прогресса;
- аудитория первой версии — 18+;
- API-ключи вводятся пользователем и не входят в исходный код или APK.

## 3. Компоненты

### Android

- `MaterialImporter` извлекает текст, страницы и структуру;
- `CurriculumEditor` управляет редактируемым деревом программы;
- `LearningScheduler` выбирает новые, просроченные и слабые понятия;
- `LessonCache` хранит проверенные пакеты для офлайн-работы;
- `MasteryPolicy` обновляет прозрачную оценку усвоения;
- `ProviderRouter` обращается к Gemini, Groq и Cloudflare Workers AI;
- `LessonQualityValidator` выполняет детерминированную проверку до AI-критика;
- `LearningTaskBridge` выдаёт образовательное упражнение существующему gate pipeline.

### Внешние провайдеры

1. Gemini — анализ документов и построение плана.
2. Groq — быстрая генерация и резерв.
3. Cloudflare Workers AI — третий резерв и критик.

Идентификаторы моделей, порядок и таймауты являются конфигурацией, а не частью доменной модели.

## 4. Безопасность ключей

Без отдельного backend ключи физически находятся на устройстве, поэтому абсолютной защиты от
владельца устройства нет. Для частной версии это принимаемый риск.

- ключ вводится пользователем;
- значение шифруется ключом Android Keystore;
- секрет не попадает в Room, DataStore backup, логи, diagnostics или crash report;
- для Cloudflare используется токен с минимально необходимыми правами;
- удаление провайдера уничтожает сохранённый секрет;
- APK остаётся работоспособным без настроенного AI за счёт локальных задач.

## 5. Поток курса по материалу

1. Пользователь выбирает файл через системный picker.
2. Приложение локально извлекает текст и вычисляет SHA-256.
3. Текст разбивается на фрагменты с привязкой к странице и разделу.
4. Поставщик выделяет понятия отдельно по фрагментам.
5. Результаты объединяются, удаляются дубликаты, строятся prerequisites.
6. Генерируется редактируемое дерево курса.
7. После подтверждения создаётся версия плана.
8. Генерируются и проверяются следующие три–пять микроуроков.
9. Исходный файл остаётся доступен только через сохранённый URI; нормализованный текст хранится
   локально и удаляется вместе с курсом.

Команды внутри импортированного текста считаются недоверенными данными и не могут менять
инструкции генератора.

## 6. Поток курса по теме

Пользователь задаёт уровень, цель, глубину, длительность и размер урока. При наличии подключённого
источника знаний план и факты получают ссылки. Иначе используется `GroundingMode.AI_KNOWLEDGE`,
который обязан показывать предупреждение в редакторе плана и каждом спорном объяснении.

## 7. Упражнения

`ActivityContent` является закрытым типизированным payload, а `ActivityKind` — стабильным
идентификатором хранения. Начальный контракт включает:

- single/multiple choice;
- true/false;
- ordering и matching;
- fill blank, numeric и short answer;
- flashcard, evidence selection и scenario;
- code output/completion/fix и mini-code;
- teach-back по рубрике.

Сначала UI реализует объективно проверяемые типы. Свободный текст и код допускаются в gate только
после появления безопасной проверки.

## 8. Усвоение и повторение

Оценка понятия находится в диапазоне 0–100. Корректный ответ повышает её, подсказки уменьшают
прирост, ошибка снижает оценку. Замена подозрительного задания нейтральна.

Понятие считается усвоенным при одновременном выполнении условий:

- оценка не ниже 80;
- не менее трёх правильных ответов;
- не менее двух форматов;
- успешные ответы как минимум в два разных дня.

Интервалы после последовательных успехов: 10 минут, 1, 3, 7, 14 и 30 дней. После ошибки повтор
назначается через 10 минут.

Начальная пропорция сессии: 50% нового, 35% повторения, 15% слабых prerequisites. Домен выбирает
понятия, а генератор отдельно подбирает формат упражнения.

## 9. Качество

Пакет сначала проходит детерминированную проверку:

- JSON/domain schema;
- уникальность вариантов;
- существование и однозначность ответа;
- корректность порядка, пар, blank markers и code tests;
- длительность микроурока;
- ссылки на источник в source-grounded режиме;
- минимальные semantic quality scores.

Затем при необходимости используется AI-критик. Последовательность восстановления:

1. один repair того же результата;
2. одна перегенерация с замечаниями;
3. следующий провайдер;
4. после трёх провайдеров — карантин и локальный fallback.

Порог: correctness и grounding ≥90, clarity и pedagogy ≥80, format validity ≥90, общий ≥85.

## 10. Надёжность и энергопотребление

- генерация запускается через WorkManager только при необходимости пополнить кэш;
- не выполняется постоянный polling;
- большие файлы разбираются потоково;
- запросы отменяются при потере сети;
- повтор 429/5xx использует backoff и circuit breaker;
- одновременно выполняется не более одного задания генерации курса;
- режим «только Wi‑Fi» доступен пользователю;
- при низком заряде фоновая генерация откладывается;
- network/AI failure не меняет состояние gate.

## 11. Граница первого блока

Первый блок содержит чистые модели, детерминированный validator, mastery policy, scheduler и
статический офлайн-урок. Он не меняет Room, навигацию, manifest или существующий monitoring
pipeline.
