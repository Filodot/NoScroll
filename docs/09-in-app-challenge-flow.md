# In-app challenge flow

Статус: принято для прототипа 0.5.0. Этот документ заменяет прежнее решение показывать
Accessibility overlay поверх YouTube во всех сценариях задания.

## Принцип

NoScrol не размещает собственное окно поверх YouTube. Accessibility Service используется
только для локального распознавания Shorts и выполнения системного действия `Back`.
Арифметическое задание, ошибки ответа, Emergency Stop и успешная разблокировка отображаются
в обычной Activity NoScrol.

## Вход в Shorts

1. Пока разрешённого интервала нет, доступ имеет состояние `LOCKED`.
2. Пользователь открывает NoScrol и получает лёгкое входное задание.
3. Правильный ответ атомарно удаляет pending task, сбрасывает счётчик интервала и сохраняет
   разрешение до `solvedAt + configuredInterval`.
4. После успеха пользователь сам нажимает «Открыть YouTube».
5. Попытка обойти этот путь прямым входом в Shorts приводит к `Back` и открытию NoScrol.

## Истечение интервала

1. При достижении `n` минут gate переходит в состояние `ARMED`; текущий Short не прерывается.
2. Первый подтверждённый `TYPE_VIEW_SCROLLED` после защитного окна 500 мс создаёт interval
   task, выполняет `Back` и открывает NoScrol.
3. Коалесцер сохраняет признак свайпа, даже если YouTube сразу прислал более новое событие
   изменения контента.
4. Если события свайпа всё же недоступны, через 2 минуты после достижения лимита действует
   принудительный fail-safe: при подтверждённом или недавно подтверждённом Shorts создаётся
   задача и выполняется выход.
5. Пока pending task существует, любая новая подтверждённая попытка открыть Shorts немедленно
   закрывается.
6. Среднее или сложное задание также выдаёт новый полный интервал `n` минут.

## Отказоустойчивость

- Pending task и разрешённый интервал сохраняются в Room.
- Перезапуск процесса не разблокирует Shorts.
- Неопределённый результат детектора не блокирует обычное видео и не считается свайпом.
- Если Android не смог автоматически вывести Activity NoScrol на передний план, Shorts всё
  равно покидаются, а задача остаётся доступной при ручном открытии приложения.
- Системная навигация принадлежит обычной Activity; отдельного overlay-window больше нет.

## Критерии приёмки

- текущий Short не закрывается только из-за достижения таймера;
- следующий подтверждённый свайп после таймера закрывает Shorts;
- задание невозможно решить внутри YouTube;
- «Решить позже» оставляет доступ закрытым;
- Back с экрана успеха не запускает YouTube;
- только явная кнопка «Открыть YouTube» запускает его после правильного ответа;
- обычные видео YouTube не вызывают Shorts task gate.
